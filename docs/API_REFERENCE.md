# API Reference

The AutoShowTracker HTTP API runs on `http://127.0.0.1:7600` by default. All endpoints return JSON. FastAPI automatically generates interactive API documentation at `/docs` (Swagger UI) and `/redoc` (ReDoc).

## Media Events

### POST /api/media-event

Receive a media playback event from the browser extension or external source.

**Request Body**:

```json
{
  "type": "play",
  "timestamp": 1710300000000,
  "tab_url": "https://www.netflix.com/watch/81511783",
  "tab_id": 42,
  "position": 125.3,
  "duration": 3420.0,
  "source": "show-tracker-content",
  "metadata": {
    "url": "https://www.netflix.com/watch/81511783",
    "url_match": {
      "platform": "netflix",
      "content_id": "81511783",
      "extra": {}
    },
    "schema": [
      {
        "@type": "TVEpisode",
        "name": "Ozymandias",
        "partOfSeason": { "seasonNumber": 5 },
        "episodeNumber": 14
      }
    ],
    "og": {
      "og:title": "Breaking Bad",
      "og:type": "video.episode"
    },
    "title": "Breaking Bad | Netflix",
    "video": [
      {
        "playing": true,
        "currentTime": 125.3,
        "duration": 3420.0,
        "src": "blob:https://www.netflix.com/...",
        "playerType": "netflix"
      }
    ]
  }
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `type` | string | Yes | Event type: `"play"`, `"pause"`, `"ended"`, `"heartbeat"`, or `"page_load"` |
| `timestamp` | integer | Yes | Unix timestamp in milliseconds |
| `tab_url` | string | No | Full URL of the browser tab |
| `tab_id` | integer | No | Chrome tab ID |
| `position` | float | No | Current playback position in seconds |
| `duration` | float | No | Total video duration in seconds |
| `source` | string | No | Event source identifier (default: `"show-tracker-content"`) |
| `metadata` | object | No | Rich metadata scraped from the page (see below) |

**Metadata Object**:

| Field | Type | Description |
|-------|------|-------------|
| `url` | string | Page URL |
| `url_match` | object or null | Result of URL pattern matching (`platform`, `content_id`, `extra`) |
| `schema` | array | JSON-LD structured data found on the page |
| `og` | object | Open Graph meta tags (key-value pairs) |
| `title` | string | Page `<title>` text |
| `video` | array | Detected `<video>` element states |

**Response** `200 OK`:

```json
{
  "status": "ok",
  "message": "Received play event"
}
```

### GET /api/currently-watching

Return the current detection state. Considers playback stale after 2 minutes without a heartbeat.

**Response** `200 OK` (actively watching):

```json
{
  "is_watching": true,
  "event_type": "heartbeat",
  "tab_url": "https://www.netflix.com/watch/81511783",
  "title": "Breaking Bad | Netflix",
  "position": 542.1,
  "duration": 3420.0,
  "last_update": 1710300060000
}
```

**Response** `200 OK` (not watching):

```json
{
  "is_watching": false,
  "event_type": null,
  "tab_url": null,
  "title": null,
  "position": null,
  "duration": null,
  "last_update": null
}
```

### POST /api/unresolved/{event_id}/resolve

Manually assign an unresolved event to a specific show/episode. Creates a watch event.

**Path Parameters**:

| Parameter | Type | Description |
|-----------|------|-------------|
| `event_id` | integer | Unresolved event ID |

**Request Body**:

```json
{
  "show_id": 1,
  "season_number": 3,
  "episode_number": 7
}
```

**Response** `200 OK`:

```json

  "status": "ok",

```

**Response** `404 Not Found`:

```json

  "detail": "Unresolved event not found"
}
```

**Request Body**:


```json
[
  {
    "tmdb_id": 2734,
    "title": "Law & Order: Special Victims Unit",
```

## Browser Extension Event Schema

The Chrome extension content script sends events to the background service worker, which enriches them with tab metadata and forwards them to `POST /api/media-event`.

### Event Types

| Type | Trigger |
|------|---------|
| `play` | A `<video>` element starts playing |
| `pause` | A `<video>` element is paused |
| `ended` | A `<video>` element finishes playback |
| `heartbeat` | Sent every 15 seconds while a video is playing |
| `page_load` | A page with a detected `<video>` element finishes loading |

### Event Flow

```
Content Script                Background Worker              Show Tracker API
     │                              │                              │
     ├── message (media event) ────►│                              │
     │                              ├── enriches with tab URL/ID   │
     │                              ├── checks tracking enabled    │
     │                              ├── POST /api/media-event ────►│
     │                              │                              ├── stores state
     │                              │◄── 200 OK ──────────────────┤
     │                              │                              │
```

### Content Script Metadata Extraction

The content script extracts metadata from the page in this order:

1. **URL pattern matching**: Checks if the URL matches known streaming platforms.
2. **JSON-LD schema**: Looks for `<script type="application/ld+json">` with `TVEpisode`, `TVSeries`, or `VideoObject` types.
3. **Open Graph tags**: Reads `<meta property="og:*">` tags for title, type, and description.
4. **Page title**: Falls back to `document.title`.
5. **Video element state**: Reads `currentTime`, `duration`, `paused`, and `src` from all `<video>` elements.

### Connection Status

The background worker periodically checks the `/api/health` endpoint (every 30 seconds) and tracks connection status (`connected`, `disconnected`, `unknown`). The popup UI displays this status.
